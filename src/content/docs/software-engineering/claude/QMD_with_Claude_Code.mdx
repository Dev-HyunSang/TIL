- [Query Markup Documents (QMD)](https://thinkingsian.com/QMD/)

## 검색의 종류와 Semantic Search

- 파일 검색 방식은 두 가지로 나눌 수 있음.
    - `Lexical search` 검색어가 포함된 것을 검색함. 윌에게 직관적으로 떠오르는 것들을 검색할 때 사용하는 방식임. `BM25`가 대표적임.
    - `Semantic search` 검색어와 의미가 비슷한 것을 검색함. 의미로 검색한다는 것엔 여러 방식이 있겠지만 `Personal Knowledge Management`에서 주로 다루는 방식에선 `embedding model`을 사용한 방식이라고 보면 됨.

## 어떻게 설정했는가?

- Claude Code 연동 규칙 (전역 CLAUDE.md)으로 설정되었음.
- 오류/버그/환경 문제를 해결하면 Claude가 트러블슈팅 노트를 자동 작성:
    - 위치: `~/Documents/HyunSang Park/Troubleshooting/YYYY-MM-DD-<slug>.md` (→ `vault` 컬렉션이 인덱싱)
    - 내용: 증상(에러 원문), 원인, 해결 방법, 환경. 본문 한국어.
- 프로젝트 개발 시 어디까지 프로젝트 개발이 되었는지 QMD를 통해서 기록하게 됨.
    - 예시로, 인증 절차의 경우 JWT 토큰 발급, 토큰 검증 등 기본적인 인증 절차가 어떻게 구현되었는지 작성되어 있음.
- Claude Code를 통해서 해당 프로젝트 내에서 Claude Code의 세션이 생성된 경우, 이러한 세션 내용도 QMD를 통해서 기록되고 있음.
