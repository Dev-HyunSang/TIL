# 2024.01.23 - DO NOT USING IOUTIL

  • Go1.16버전부터 io/ioutl를 사용할 수 없게 되었음.
  • Go 1.16 Release Notes에서 왜 io/ioutl를 사용할 수 없는지 아래와 같이 설명하고 있음.
  • Viusal Studio Code 상에서 io/ioutl를 사용하게 되면, 취소선을 표시하여 사용할 수 없는 패키지라는 점을 알리고 있음.
  • 업데이트가 되고 있지 않은 패키지임으로 사용하면 안 됨. 보안 취약점이 존재할 수도 있음.

변경된 점:

  • Discard => io.Discard
  • NopCloser => io.NopCloser
  • ReadAll => io.ReadAll
  • ReadDir => os.ReadDir (note: returns a slice of os.DirEntry rather than a slice of fs.FileInfo, fs.FileInfo 조각이 아닌 os.DirEntry 조각을 반환합니다)
  • ReadFile => os.ReadFile
  • TempDir => os.MkdirTemp
  • TempFile => os.CreateTemp
  • WriteFile => os.WriteFile

The io/ioutil package has turned out to be a poorly defined and hard to understand collection of things. All functionality provided by the package has been moved to other packages. The io/ioutil package remains and will continue to work as before, but we encourage new code to use the new definitions in the io and os packages.
io/ioutil 패키지는 잘 정의되지 않았고 사물 모음을 이해하기 어려운 것으로 드러났습니다. 패키지에서 제공하는 모든 기능은 다른 패키지로 이동되었습니다. io/ioutil 패키지는 남아 있으며 이전과 같이 계속 작동하지만 새로운 코드가 io 및 os 패키지의 새로운 정의를 사용하도록 권장합니다.
